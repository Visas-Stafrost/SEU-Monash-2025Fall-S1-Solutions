Chapter 4: Nonparametric Techniques

4.1 Introduction

Potential problems for parametric techniques:

  • The assumed parametric form may not fit the ground-truth density encountered in practice, e.g:
    • Assumed parametric form: Unimodal(such as Gaussian pdf)
    • Ground-truth density: Multimodal(such as mixture of Gaussian pdfs)

So here comes the nonparametric techniques: p(x∣wj)p(x|w_j)p(x∣wj​) doesn't have parametric form. We use the data to speak for themselves.

  1. Parzen Windows
  2. knk_nkn​-nearest-neighbor

4.2 Probability Density Estimation

General Settings

  • Feature space: F=Rd\mathcal{F} = \mathbf{R}^dF=Rd
  • Feature vector: x∈Fx \in \mathcal{F}x∈F
  • pdf function: x∼p(⋅)x \sim p(\cdot)x∼p(⋅)

Fundamental Fact: The probability (a smoothed/averaged version of p(x)p(x)p(x)) of a vector xxx falling into a region R⊂F\mathcal{R} \subset \mathcal{F}R⊂F:

P=Pr[x∈R]=∫Rp(x′)dx′P = Pr[x \in R] = \int_{\mathcal{R}} p(x^{'}) dx^{'} P=Pr[x∈R]=∫R​p(x′)dx′

Given n examples (i.i.d.)

x1,x2,⋯ ,xnx_1, x_2, \cdots, x_nx1​,x2​,⋯,xn​

with xi∼p(⋅)(1≤i≤n)x_i \sim p(\cdot) \quad (1 \leq i \leq n)xi​∼p(⋅)(1≤i≤n)

Let XXX be the (discrete) random variable representing the number of examples falling into R\mathcal{R}R, so X will take Binomial Distribution: X∼B(n,P)X \sim \mathcal{B}(n, P)X∼B(n,P) and we have:

Pr[X=r]=(nr)Pr(1−P)n−rE[X]=nP⇒P=E[X]nPr[X = r] = \binom{n}{r} P^r (1-P)^{n-r} \\ \mathcal{E}[X] = nP \quad \Rightarrow \quad P = \frac{\mathcal{E}[X]}{n} Pr[X=r]=(rn​)Pr(1−P)n−rE[X]=nP⇒P=nE[X]​

If we assume that p(x)p(x)p(x) is constant and region R\mathcal{R}R is so small that p(⋅)p(\cdot)p(⋅) hardly varies with R\mathcal{R}R, then we have:

P=Pr(x∈R)=∫Rp(x′)dx′≈p(x)∫R1dx′=p(x)VP = Pr(x \in \mathcal{R}) = \int_R p(x^{'})dx^{'} \approx p(x) \int_{\mathcal{R}} 1 dx^{'} = p(x)V P=Pr(x∈R)=∫R​p(x′)dx′≈p(x)∫R​1dx′=p(x)V

  • xxx is a point in R\mathcal{R}R
  • VVV is the volume enclosed by R\mathcal{R}R
  • k/nk/nk/n: success / total
  • all curve converge to a single peak at E(X)\mathcal{E}(X)E(X)
  • when n get larger, curve gets sharper around E(X)\mathcal{E}(X)E(X)

XXX peaks sharply about E(X)\mathcal{E}(X)E(X) when n is large enough.

Let kkk be the actual value of XXX after observing the i.i.d. training examples x1,x2,⋯ ,xn{x_1, x_2, \cdots, x_n}x1​,x2​,⋯,xn​

Given those evidence, we have k≈E[X]k \approx \mathcal{E}[X]k≈E[X], hence we can conclude that:

p(x)=k/nVp(x) = \frac{k/n}{V} p(x)=Vk/n​

If we fixed the value of VVV and are able to get more training examples, then the value of k/nk/nk/n will converge to the true value of p(x)p(x)p(x), which is a smoothed version:

PV=∫Rp(x′)dx′∫Rdx′\frac{P}{V} = \frac{\int_{\mathcal{R}}p(x^{'})dx^{'}}{\int_{\mathcal{R}}dx^{'}} VP​=∫R​dx′∫R​p(x′)dx′​

However, we hope to estimate p(x)p(x)p(x) which is not a smoothed version, so we need to make VVV approach 0, which lead to the following problems:

  • V→0V \rightarrow 0V→0, the region R\mathcal{R}R will become so small that there my be no training examples falling into the region, which makes no sense.
  • If 1 or 2 sample accidentally fall into the region R\mathcal{R}R, then the estimation will become infinite, which is also not reasonable.

Theoretically, we construct a series fo regions R1,R2,⋯ ,Rm\mathcal{R}_1, \mathcal{R}_2, \cdots, \mathcal{R}_mR1​,R2​,⋯,Rm​ which contain the dot xxx to overcome the limitations. The first region use 1 sample, the second region use 2 samples, and so on. Hence we have:

pn(x)=kn/nVnp_n(x) = \frac{k_n/n}{V_n} pn​(x)=Vn​kn​/n​

  • VnV_nVn​: the volume of region Rn\mathcal{R}_nRn​
  • nnn: the number of training examples
  • knk_nkn​: the number of training examples falling into Rn\mathcal{R}_nRn​

Requirements to be satisfied, which make pn(x)p_n(x)pn​(x) converge to p(x)p(x)p(x):

  1. lim⁡n→∞Vn=0\lim_{n \to \infty} V_n = 0limn→∞​Vn​=0: ensure the space-smoothed P/VP/VP/V converge to p(x)p(x)p(x) under the situation of uniform convergence in the region and continuity of p(⋅)p(\cdot)p(⋅) at point xxx.
  2. lim⁡n→∞kn=∞\lim_{n \to \infty} k_n = \inftylimn→∞​kn​=∞: meaningful only when p(x)≠0p(x) \neq 0p(x)=0 and guarantees that the ratio of frequencies can converge to the probability PPP
  3. lim⁡n→∞kn/n=0\lim_{n \to \infty} k_n/n = 0limn→∞​kn​/n=0

Methods:

  1. Parzen Windows: Fix VnV_nVn​ and determine knk_nkn​
  2. knk_nkn​-nearest-neighbor: Fix knk_nkn​ and determine VnV_nVn​

4.3 Parzen Windows

Assumptions:

  • Rn\mathcal{R_n}Rn​: a d-dimensional hypercube
  • hnh_nhn​: the length of the edge of Rn\mathcal{R_n}Rn​

Determine knk_nkn​ with window function, a.k.a. kernel function, potential function, etc.

φ(x−xihn)=1⇔\varphi(\frac{x-x_i}{h_n}) = 1 \quad \Leftrightarrow \quadφ(hn​x−xi​​)=1⇔ xix_ixi​ falls within the hypercube of volume VnV_nVn​ centered at xxx with edge length hnh_nhn​

  • x−xix - x_ix−xi​: the relative position between the sample point xix_ixi​ and the target piont xxx
  • hnh_nhn​: bandwidth parameter which controls the width of the window

pn(x)=kn/nVn=1n∑i=1n1Vnφ(x−xihn)p_n(x) = \frac{k_n/n}{V_n} = \frac{1}{n} \sum_{i=1}^n \frac{1}{V_n} \varphi(\frac{x-x_i}{h_n}) pn​(x)=Vn​kn​/n​=n1​i=1∑n​Vn​1​φ(hn​x−xi​​)

  • meaning: the average contribution of the sample points to the target point xxx
  • The kernel function φ\varphiφ weights the influence of each sample point xix_ixi​ on xxx:
    1. If xix_ixi​ is very close to xxx (i.e., ∣x−xi∣|x - x_i|∣x−xi​∣ is small), then φ(x−xihn)\varphi (\frac{x - x_i}{h_n})φ(hn​x−xi​​) will approach 1.

  • Notice: xxx is a multi-dimensional vector and the window bandwidth hdh_dhd​ do the same for all the dimensions. When using the substitution u=x−xihn\mathbf{u}=\frac{x-x_i}{h_n}u=hn​x−xi​​, we get dx=hnddudx = h_n^d d\mathbf{u}dx=hnd​du

Above all we can conclude that:

  • windows function (being pdf) φ(⋅)\varphi(\cdot)φ(⋅) + window width hnh_nhn​ + training data xix_ixi​= Parzen pdf pn(⋅)p_n(\cdot)pn​(⋅)

Now we exploit the effect of window width hnh_nhn​ on the Parzen pdf pn(x)p_n(x)pn​(x). We define function δn(x)\delta_n(x)δn​(x) as:

We can rewrite pn(x)p_n(x)pn​(x) as:

pn(x)=1n∑i=1nδn(x−xi)p_n(x) = \frac{1}{n} \sum_{i=1}^n \delta_n(x - x_i) pn​(x)=n1​i=1∑n​δn​(x−xi​)

  • pn(x)p_n(x)pn​(x): superposition of n sample contributions interpolations

  • xix_ixi​: contributes to pn(x)p_n(x)pn​(x) based on the distance between xxx and xix_ixi​

  • Influence:

    • hnh_nhn​ very large
      1. δn(x)\delta_n(x)δn​(x) being board with small amplitude
      2. pn(x)p_n(x)pn​(x) will be the superposition of n board, slowly changing functions, i.e. being smooth with low resolution
      3. may cause underfitting and be unable to capture local structure
    • hnh_nhn​ very small
      1. δn(x)\delta_n(x)δn​(x) being sharp with large amplitude
      2. pn(x)p_n(x)pn​(x) will be the superposition of n sharp pulses, i.e. being variable/unstable with high resolution

A compromised value of hnh_nhn​ should be sought for limited number of training examples.

4.4 knk_nkn​-nearest-neighbor

pn(x)=kn/nVnp_n(x) = \frac{k_n/n}{V_n} pn​(x)=Vn​kn​/n​

  • Fix knk_nkn​ then determine VnV_nVn​
  • Procedure: specify kn→k_n \rightarrowkn​→ centere a cell about x→x \rightarrowx→ grow the cell until capturing knk_nkn​ nearest exmaples →\rightarrow→ return cell volumes as VnV_nVn​
  • The principled rule to specify knk_nkn​ (the sufficient and necessary conditions of pn(x)p_n(x)pn​(x) converging to p(x)p(x)p(x)):
    1. limn→∞kn=∞lim_{n \to \infin} k_n = \infinlimn→∞​kn​=∞
    2. limn→∞knn=0lim_{n \to \infin} \frac{k_n}{n} = 0limn→∞​nkn​​=0
  • A rule-of-thumb (经验法则) choice for knk_nkn​: kn=nk_n = \sqrt{n}kn​=n​

  • 1-dimension example
    • Smaller knk_nkn​ produces detailed but noisier estimates
    • Larger knk_nkn​ creates smoother estimates but with reduced local detail.
  • 2-dimension example
    • The kn=5k_n = 5kn​=5 estimation dynamically adapts to the data density, showing higher values in dense areas and lower values in sparse regions.

4.5 Nearest-Neighbor Rule

4.5.1 Definition

Given:

  • The label space: Ω={ω1,ω2,…,ωc}\Omega = \{\omega_1, \omega_2, \dots, \omega_c\}Ω={ω1​,ω2​,…,ωc​}
  • A set of nnn labeled training examples: Dn={(xi,θi) ∣ 1≤i≤n}\mathcal{D}^n = \{(\mathbf{x}_i, \theta_i) \ | \ 1 \leq i \leq n\}Dn={(xi​,θi​) ∣ 1≤i≤n}, where:
    • xi∈Rd\mathbf{x}_i \in \mathbb{R}^dxi​∈Rd: The iii-th feature vector in ddd-dimensional space
    • θi∈Ω\theta_i \in \Omegaθi​∈Ω: The label associated with xi\mathbf{x}_ixi​

For a test example x\mathbf{x}x:

  1. Identify its nearest neighbor x′\mathbf{x}'x′: x′=arg⁡min⁡xi∈{x1,…,xn}D(xi,x)\mathbf{x}' = \arg\min_{\mathbf{x}_i \in \{\mathbf{x}_1, \dots, \mathbf{x}_n\}} D(\mathbf{x}_i, \mathbf{x})x′=argminxi​∈{x1​,…,xn​}​D(xi​,x),
    • D(a,b)D(\mathbf{a}, \mathbf{b})D(a,b): distance metric between two vectors a\mathbf{a}a and b\mathbf{b}b, such as the Euclidean distance.
  2. Assign the label θ′\theta'θ′ associated with x′\mathbf{x}'x′ to x\mathbf{x}x.

4.4.2 Voronoi tessellation (维诺图)

Each training example x\mathbf{x}x leads to a cell in the Voronoi tessellation

  • any point in the cell is closer to x\mathbf{x}x than to any other training examples
  • partition the feature space into n cells
  • any point in the cell share the same class label as x\mathbf{x}x

4.4.3 Error Rate of Nearest Neighbor Rule

P(e)=∫P(e∣x)p(x)dxP(e) = \int P(e|x) p(x) dx P(e)=∫P(e∣x)p(x)dx

  • P(e)P(e)P(e): The average probability of error based on nearest-neighbor rule
  • P(e∣x)P(e∣x)P(e∣x): The probability of making an errorneous classification on xxx based on nearest-neighbor rule
  • p(x)p(x)p(x): The probability density function of the feature space, which describes how data points are distributed

According to the Bayes Decision Rule, we can minimize the error rate at each point xxx by choosing the class label wmw_mwm​ with the maximum posterior probability:

P∗(e∣x)=1−P(wm∣x)P^*(e|x) = 1 - P(w_m|x) P∗(e∣x)=1−P(wm​∣x)

  • P(wm∣x)P(w_m|x)P(wm​∣x): the posterior probability of the most likely class at xxx, i.e. P(wm∣x)=max⁡1≤i≤cP(wi∣x)P(w_m|x) = \max_{1 \leq i \leq c} P(w_i|x)P(wm​∣x)=max1≤i≤c​P(wi​∣x)

The Bayes risk(under zero-one loss) can be written as:

P∗(e)=∫P∗(e∣x)p(x)dxP^*(e) = \int P^*(e|x) p(x) dx P∗(e)=∫P∗(e∣x)p(x)dx

Proof of the lower bound:

P∗(e)P^*(e)P∗(e) is the minimum possible value of P(e)P(e)P(e) given by the Bayesian decision rule, which is the optimal classifier.

Proof of the upper bound:

∑i=1cP2(wi∣x)=P2(wm∣x)+∑i≠mP2(wi∣x)\sum_{i = 1}^c P^2(w_i | x) = P^2(w_m|x) + \sum_{i \not ={m}} P^2(w_i|x) i=1∑c​P2(wi​∣x)=P2(wm​∣x)+i=m∑​P2(wi​∣x)

    • P(wi∣x)≥0P(w_i | x) \geq 0P(wi​∣x)≥0

4.4.6 Computational Complexity of KNN Rule

Given nnn labeled training examples in ddd-dimensional space, the computational complexity of classifying one test example is O(dn)O(dn)O(dn)

General ways of reducing computational burden:

  • Calculate Partial Distance: Dr(a,b)=(∑j=1r(aj−bj)2)12 (r<d)D_r(a, b) = \Big(\sum_{j = 1}^r(a_j - b_j)^2 \Big)^{\frac{1}{2}} \, (r < d)Dr​(a,b)=(∑j=1r​(aj​−bj​)2)21​(r<d)
    • Using a subset r\mathcal{r}r of the total d-dimensional space
    • As more dimensions are gradually added, the values of partial distances are strictly non-decreasing
  • Pre-structing: create some form of search tree, where nearest neighbors are recursively identified following the tree structure (each root node corresponds to a brunch of the disciminant region)
    • This may cause error because when the testing sample is near the boundary of the region, the nearest neighbor may not be found in the discriminated region.
  • Editing/Pruning/Condensing: eliminate "redundant" ("useless") examples from the training set, e.g. example surrounded by training examples of the same class label

4.5 Distance Metric

4.5.1 Properties of Distance Metric

The NN/kNN rule depends on the use of distance metric to identify nearest neighbors.

Four properties of distance metric:

  • Non-negativity: D(a,b)≥0D(a, b) \geq 0D(a,b)≥0
  • Reflexivity (自反性): D(a,b)=0⇔a=bD(a, b) = 0 \Leftrightarrow a = bD(a,b)=0⇔a=b
  • Symmetry: D(a,b)=D(b,a)D(a, b) = D(b, a)D(a,b)=D(b,a)
  • Triangle Inequality: D(a,b)+D(b,c)≥D(a,c)D(a, b) + D(b, c) \geq D(a, c)D(a,b)+D(b,c)≥D(a,c)
