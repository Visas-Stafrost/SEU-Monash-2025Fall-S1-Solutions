# 课程介绍

本课程编号为 02 ，这是第一门结束的课，本课程无考试。

## 授课教师

2025秋季学期任课教师为**吴天星**老师。

- **老师介绍**: 本身就是干知识图谱的，属于学院漆桂林老师的高徒，推荐资料甚至夹带私货用他们组写的书。学术实力强劲并且作为年轻一代在学院有一些职务，有点拽拽的，想给人营造一种学术精英的感觉。虽然确实也是，但是相较于其他实力不弱的老师，没有那么谦逊。
- **教学风格**: 课堂上相当严肃，老师主观上就想表现出认真负责的感觉，会营造一些压迫感，所以不算很讨喜。不过我个人上课都是坐第一排占着插座拿个电脑远控主机搞项目甚至玩战旗，也没啥事，老师没下过讲台，就是两江院的网太差了得自己带个随身 WiFi 或者手机热点 + 充电宝。

## 考勤情况

- 会点人回答问题，但其实还好。因为有主动上台或者回答问题会加分的机制，真的没人上去做才会点。整个学期那种全体点名大概三四次吧，看见人少比较多就会点。其中包括倒数第二次课，明明临近期末上课内容结束，是小组展示的时候也点了，所以不要抱有侥幸心理。总结就是来捧场就行，干什么不太管。

## 课程内容

本来看课程名还以为是 Web 开发，结果是 KG 课程。算是给知识图谱方向的一个科普课（也可能是这个方向本身很多东西讲开了就是比较浅显，总之没什么理解难度），讲的比较全面，干货还行。课堂上点人回答问题一也主要是为了强制让大伙听讲，一般就是先介绍一个框架，讲了五分钟，然后找一个人上来依葫芦画瓢画一下刚才讲的东西这样，也有一些开放性小问题比如有啥问题如何改进之类。

## 作业情况

作业分为`小组 pre `和`小组课程报告`，均是小组形式，一个小组5-6个人一块完成这两个作业。

### 小组 pre

- **作业时间**: 大概会在倒数三四节的课的时候进行这个事，会提前两三周发一些论文读。论文就是选自一些老师认为的知识图谱领域的高质量论文，差不多分为补全、推理、置信度三个方向，有难有易。
- **作业内容**: 每个小组阅读论文准备 presentation ，全英文 pre （ 20 - 25 分钟），不允许全程看稿（不然 pre 过程中会说“不是让你们上来读稿子的”），不低于 20 分钟， 5 - 10 分钟 QA （全英文），会问的比较细节。pre 推荐上台人数 3 到 4 人，上台人数太多或者太少都会被讲“不是让每个人都上来”、“给我一种就一个人干活的错觉”，做了复现的话 pre 会加分。比组会还组会的既视感。

### 小组课程报告

最后一节课结课交。就是对本小组讲解的论文、本课程的内容写一篇报告，没给格式要求，横向比较鼓励互卷。属于是一个揣测圣意的作业,私以为能自圆其说就行。工作量按原话讲是“只写十几页肯定不行”，笔者所在的组拿 latex 用学术会议投稿模板搞了大概三十四页的样子。

## 仓库介绍

课程作业具体小组干了什么这里就不放了，因为是个公共的作业，知识产权属于集体。而且实际上也没什么参考价值，因为规则不透明，笔者搞的也不一定就是对的，只能按照感受给出上边的描述。`paper`下是论文，`ppt`是上课用ppt，`textbook`是老师推荐资料。
